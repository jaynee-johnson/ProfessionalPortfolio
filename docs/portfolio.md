# Technical Writing Portfolio - Project Explanations

| Project | Description |
|---------|-------------|
| [Routing Documentation](routing/routing.md) | <p>Originally authored in MadCap Flare and converted to Markdown to demonstrate a docs-as-code approach. Despite routing being core to the NICE CXone product, no dedicated routing documentation existed for many years. Processes and settings were scattered throughout the application-based online help. I identified this gap, consolidated content from across the help system into a cohesive Routing section, and used snippets to single-source it so content could live both here and alongside its relevant application.</p><p>I wrote the [Routing](routing/routing.md), [Dynamic Delivery](routing/dynamicdelivery/dynamicdelivery.md), and [Static Delivery](routing/staticdelivery/staticdelivery.md) overviews and the [Dynamic Delivery enablement page](routing/dynamicdelivery/enabledynamicdelivery.md). The remaining sections are a combination of my own writing and content pulled from existing documentation.</p> |
| [Release Notes Archive](releaseNotesArchive/index.htm) | <p>An archive of NICE CXone release notes I created to address recurring requests for historical release documentation. Because the online help only surfaced current releases, internal and external users frequently needed older versions sent manually. I built this archive to enable self-service access, structuring it as a dedicated Flare project using the online help template I helped design and build.</p><p> Note that release notes are a compilation of contributions from the entire writing team. I owned the process and final output, but did not write all content. The design reflects the updated brand colors and fonts NICE introduced in 2025.</p> |
