How to Use This Page

All new products and features are shown by default. If you want to see only products and features in your CXone system, you can clear the checkboxes to remove features you don't need from your display.

Each feature description includes a table that states whether the following details apply to it. means that the detail applies to the feature and means that the detail does not apply to the feature.

Customer Request The feature was developed by customer request.
UI Change The feature introduces major user interface (UI) changes.
Availability

  • On toggle— This is a minor release. The feature should be toggled on at the time of deployment. If you've registered for Service Notifications , watch for one that indicates when that is.
  • On deploy—The feature is immediately available when the new release deploys to your environment.

New and Returning Visitors Rule Condition

Use the New and Returning Visitors condition when creating CXone Guide engagement rules. With this condition, you can display proactive offers or templates to visitors based on their visit history:

  • First time visitors: Display proactive offers or templates that are specific to first-time visitors. For example, you can display a proactive offer that includes a welcome message for new visitors.

  • Returning visitors: Display proactive offers or templates that are specific to returning visitors. You can specify how many times they visited within a certain time period. For example, you can display a proactive offer specifically for returning users who visited three or more times within the past 14 days.

CXone Studio

These release notes are for a product that's part of an Out-of-Band Controlled Release (CR) program following a CI/CD methodology. The changes indicated in these release notes have already been made to the product.

Preview Features

21 November 2023

Improvements for Actions that Open Other Scripts

The following improvements have been made to the appearance and functionality of action icons for actions that open other scripts, such as RUNSUB. When placed on the script canvas, these actions now:

  • Have a unique shape. The new shape is a rectangle that's the same width as other actions, but taller.
  • Show two lines of the most important property. For example, they may show the name of the script they're configured to open.
  • Include a clickable icon that opens the other script in a new browser tab. The icon appears when hovering the cursor over the action icon. If the other script is already open, focus shifts to that script.
Customer Request UI Change Availability Controlled Release

Action Icons No Longer Highlighted when Opening Script

Previously, when opening an existing script, all the actions were highlighted on the canvas. This required clicking in the white space of the canvas to deselect the icons prior to working with the script. This behavior has been fixed. When opening an existing script, no actions are automatically highlighted.

Customer Request UI Change Availability Controlled Release

Text and Icon Alignment Fixed

Previously, in the menu to the left of the All Scripts page, the icon and the Scripts label were not aligned. Now, the bottom of the icon and the text are properly aligned.

Customer Request UI Change Availability Controlled Release

Improvements to Breadcrumb Links on All Scripts Page

Previously, it was not obvious that the text at the top of the All Scripts page was really breadcrumb links. Now, the appearance of this text has been changed. The new format makes it more obvious that the text is breadcrumb links that can be clicked to navigate backward in the folder structure.

Customer Request UI Change Availability Controlled Release

Improvements to Script Deactivation

The following improvements have been made to script deactivation: 

  • Previously, when a script was deactivated, it still appeared in the list of active scripts on the All Scripts page. Now, when a script is deactivated, it disappears from this page. Deactivated scripts can still be viewed by selecting All or Inactive from the drop-down on the top right side of the All Scripts page.
  • The Deactivate option can be found in the options menu for scripts listed on the All Scripts page and from the drop-down that appears on the script canvas page when clicking the name of an unlocked, read-only script. This option now only appears for active scripts. It's hidden for inactive scripts.
Customer Request UI Change Availability Controlled Release

Scripts in View-Only Mode Cannot be Modified

When a script is locked, it's put into view-only mode. Any other Studio users who open a locked script cannot make any changes to it.

Customer Request UI Change Availability Controlled Release

Errors Related to a Specific Action are Deleted When the Action is Removed

Previously, if a specific action caused an error and that action was then removed from the script, the next action added to the script inherited the error from the deleted action. Now, when an action is removed from the script, all errors associated with it are also removed.

Customer Request UI Change Availability Controlled Release

Update to Fit to Screen Option

Previously, when the Actions palette was open and the Fit to Screen button was clicked, the actions on the left side of the canvas were hidden behind the Actions palette. Now when this button is clicked, the canvas fits to the space outside of the Actions palette, so that all actions remain visible on the canvas.

Customer Request UI Change Availability Controlled Release

Sequence Editor Improvements: 

The following improvements have been made to the Sequence editor for actions such as PLAY:

  • Previously, when using the Sequence editor, the cursor did not automatically appear in the text field after clicking the Text to Speech option. Now, the cursor appears in the text field after clicking Text to Speech so users can start typing immediately.
  • Previously, when all sequences were deleted from the Sequence editor and the changes were applied, the action's Phrase property was also deleted. Now, the contents of the Phrase property is not touched when saving changes after deleting all contents of the Sequence property.
Customer Request UI Change Availability Controlled Release

ASSIGN Keyword Accepts Values that Start with a Pound Sign ( # )

Previously, when using the ASSIGN keyword in a SNIPPET action, values that started with a pound sign ( # ) caused an error. Now, you can assign a value starting with this symbol to a variable using the ASSIGN keyword.

Customer Request UI Change Availability Controlled Release

SELECT Statement Syntax Improvements

The syntax for SELECT statements in SNIPPET actions has been improved. The editor now requires at least one CASE statement. When multiple CASE statements are added to a SELECT, it also requires a DEFAULT statement. An error is caused if DEFAULT is the only statement, or if multiple CASE statements are added without a DEFAULT.

Customer Request UI Change Availability Controlled Release

Syntax Highlighting in the SNIPPET Editor

Code added in the SNIPPET action editor window is now formatted with a syntax highlighter. Different colors are used for commands and keywords, objects, functions, operators, parentheses and brackets, and so on. This makes code easier to read.

Cloud Transcription Service Renamed and Moved

CXone Cloud Transcription Service has been renamed to Turn-by-Turn Transcription. It has also been moved under the new Transcription Hub menu option in the Automation and AI section in CXone. The new name reflects the way this transcription service is used in CXone.

Customer Request UI Change Availability On deploy This feature is immediately available when the new release deploys to your environment.

Features Removed from This Release

This section describes product changes that were included in the initial coming soon announcement and have since been removed from this release. These will not be part of the December 2023 Minor release but may be part of a future release.

Removed Applications and Features

Dashboard Application Replaced

The current Dashboard application will be removed from CXone. You will no longer be able to access or use it. Your system will migrate to the new CXone Dashboard application for a similar but improved experience. The new CXone Dashboard doesn't currently support FedRAMP. Until it does, FedRAMP-enabled systems will include the current Dashboard application but not the new CXone Dashboard.
