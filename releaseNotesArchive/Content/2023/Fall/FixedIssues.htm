Cumulative Update 5

Published 2 January 2024*

ACD

  • Hours of operation frequently threw exceptions when the browser was set to a language other than English. (AAD-23744)

  • In some dynamic delivery routing environments, agents would set their state to Available and immediately have it change to Unavailable - Refused. (AOR-4138)

  • When an invalid contact was routed, the assigned agent was automatically placed in a Refused state. (AOR-4214)

  • Interactions where the contact hung up in the IVR sometimes produced recordings that were several hours long. (ORC-28194)

  • The Workflow Data page sometimes showed a "rate exceeded" error that resulted in users not being able to make updates to Workflow Data. (AAD-24028)

  • Workflow Data pages sometimes returned a "bad request" error when users tried to make updates. (AAD-23959)

  • When a user's name included Japanese characters, the ACD screen didn't display correctly. (AAD-23475)

  • The SetAgentStudio action allowed you to set a supervisor’s CXone Agent state to Available while they were actively monitoring agents in CXone Supervisor. When a supervisor is Available and receives a call while they’re monitoring, the supervisor’s call merges with the call of the agent they’re monitoring, coaching, barging, or taking over. SetAgent can no longer change a supervisor’s agent state to Available while they’re monitoring agents. It takes the On Error branch instead. You can use the On Error branch to redirect the contact; for example, to voicemail or to another agent or skill. Alternatively, you can add logic to your script that bypasses the SetAgent to Available logic if the supervisor’s agent state is the system Unavailable state or a custom Unavailable state used specifically for monitoring. Your script must then reroute the contact. The SetAgent page contains more information about preventing this issue. (ORC-28585)

API

  • In v27, GET/agents/state-history took a long time to execute and sometimes timed out. (RPT-34680)

CXone Agent

  • Commitments were scheduled for the default business unit time zone instead of the selected time zone. (AW-18956)

  • When an agent refused an inbound call and returned to Available shortly after, their state changed to Working and they couldn't receive new calls. (AW-19491)

  • Custom workspace URLs couldn't access audio or video hardware. (AW-19410)

Reporting

  • In report schedules, sometimes the schedule recurrence settings weren't saved correctly. (RPT-34012)

  • In the Abandon Rate Compliance report, the Answered column always showed a value of 0. (RPT-34627)

Studio

  • In some linked script scenarios, the child script didn't return to the parent script. (ORC-28600)

  • Rest Api Action took the error branch while executing any REST API endpoint URL. (ORC-29169)

CXone Workforce Management

  • Data may be missing when you export the WFM Adherence report during the first half of an hour. For example, if you export the Adherence report between 10:00 and 10:29, the report may not include data for all days selected in the report prompt. To avoid this problem, export report data during the second half of the hour. For example, export reports during 10:30–11:00, 11:30–12:00, and so on. A fix for this issue is currently in progress.
