Global Changes

Configurable Toast Message Duration

All users can adjust the length of time that toast messages appear on their screen. This setting is in a new Accessibility tab in My Profile. Users can configure the messages to stay on-screen for a longer period of time.

Toast messages are small, temporary pop-up messages on the bottom of the screen indicating things like errors or successes. They are built in to CXone functionality and can appear on any screen.

Users will receive a timeout warning after 20 minutes of inactivity in CXone. This update does not apply to agent applications. Users can choose to stay logged in or to log out.

Support Added for Claim-Based OpenID Connect Linking of New Users

Some ID providers require a unique subject identifier for each CXone application. This has made it impossible to pre-configure federated identity for these users in CXone. Instead, the user had to log in using a different method, and then manually link their OpenID Connect identity to their user account.

With this release, CXone can use a different claim value, like an email address, to establish the user's identity at their first login only. CXone then automatically switches to the unique OpenID Connect subject identifier. This update makes it easier to set up a user's federated identity using the unique, stable OpenID Connect identifier.

ACD

Assigned Inactive State for User Skill Assignment Added to ACD Bulk Upload Template

In the user interface, you can set user skill assignments to one of the following three states:

Assigned active—the skill is assigned to the user, and CXone can route contacts to them based on that skill.

Assigned inactive—the skill is assigned to the user, but CXone cannot route contacts to them based on that skill.

Unassigned—the skill is not assigned to the user.

In the ACD Users bulk upload template, only the assigned active and unassigned states were supported. With this release, you can set user skill assignments to assigned inactive with the bulk upload templates.

The ACD Users bulk upload template has changed in the following ways:

  • The Skill ID column has been renamed Assigned Skills.

  • The formatting in the Assigned Skills column has been changed.

    • The capability to assign inactive skills was added. You can now enter TRUE to assign an active skill to a user. Or, you can enter FALSE to assign an inactive skill to a user.

    • The delimiters were changed to provide more consistency between the various ACD Bulk upload template formats. Pipes were changed to semicolons. Semicolons are still used the same way.

    • A semicolon has been added to the end of the format. The semicolon is present upon download, but it is optional during uploads and can be removed without affecting functionality. This change improves consistency in the system.

The previously accepted format was:

  • Skill_ID|Proficiency;Skill_ID|Proficiency;Skill_ID|Proficiency

    • For example: 197642|5;197648|1;391021|10

The new format is:

  • Skill_ID:Proficiency:Active;Skill_ID:Proficiency:Inactive;Skill_ID:Proficiency:Active;

    • For example: 197642:5:TRUE;197648:1:FALSE;391021:10:TRUE;

The Skills bulk upload template has also changed. You can now assign inactive skills with the Assigned User column. You can enter TRUE to assign an active skill to a user. Or, you can enter FALSE to assign an inactive skill to a user.

The previously accepted format was:

  • User_ID:Proficiency;User_ID:Proficiency;User_ID:Proficiency;

    • For example: 289994:5;2547173:1;2628493:10;

The new format is:

  • User_ID:Proficiency:Active;User_ID:Proficiency:Active;User_ID:Proficiency:Active;

    • For example: 289994:5:TRUE;2547173:1:FALSE;2628493:10:TRUE;

These new formats will be present when you download filled ACD bulk upload templates. This update more closely aligns the bulk upload templates to the user interface, which allows for a more seamless user experience.

The new user skill assignment format is the only acceptable format moving forward. Any future changes or additions in user skill assignments using the ACD bulk upload templates must adhere to the new format to function properly. The previous format will no longer be supported because it produced problematic and unintuitive results.

You can integrate Google Dialogflow CX with CXone in Virtual Agent Hub.

CXone Workforce Management

Overtime Activity Codes Enhancement for Calculating Adherence

You will be able to improve monitoring and reviewing overtime activities. This will be done by enabling the Overtime attribute in the overtime activity code settings. When enabling the overtime attribute, the activities codes will be mapped to all open activities.

When agents work overtime and are open for interactions or involved in one, they will be in adherence to their schedule.

Forecasting: Multiple What Ifs

You will have the option to create more than one alternate forecast to your primary forecast. This will give you the option to see how multiple changes to the forecast data affect your staffing.

With multiple alternative forecasts, you will be able to:

  • Rename each one and save them as a separate staffing plan.

  • Compare them with the primary forecast.

  • Delete all alternative forecasts with one button.

Forecasting by the Week

Forecasting by week allows generating the forecast for short time intervals. There are three additional time interval options available:

  • One week

  • Two weeks

  • Three weeks

The new options give better control over generating an accurate forecast. Values and graphs are available at the weekly level. Weekly forecasts have improved alignment with the schedule generation process.
